# Frontend (Web SPA)
> Last validated: 2026-05-22 (re-validated after hardening API error surfacing: `apiFetch` now reads JSON bodies on non-2xx; `apiRegister`/`apiResetPassword` prefer Laravel field-errors over the generic 422 message; `apiForgotPassword` throws on 429/5xx; `Account.tsx`'s explicit Sign Out also calls `apiLogout` to revoke server-side; `EmailConfirm.tsx` now shows a "Confirm change" button instead of auto-firing on mount — defeats email-link prefetchers (Outlook Safe Links, Mimecast, GSuite) and eliminates the old useEffect-on-token-hydrate race; `apiConfirmEmailChange` POSTs `{id, token}` against the new POST `/account/email/confirm` route, and refreshes the in-app user via `login(…)` so the new email shows on `/account` without a reload; `apiFetch` now accepts a `silentOn401` option so background reads on lazy pages can fail without yanking the user out — `Account.tsx`'s `fetchBillingStatus` uses it and shows the error inline instead of triggering global auto-logout; `Reset.tsx` calls `fetchMe()` after a successful reset and threads `subscribed` through `login()` so previously-subscribed users don't transit `/subscribe` while `AuthSync` catches up)

The parent-facing frontend is a **Vite + React 19 + TypeScript** single-page app (`react-router-dom` v7) under `frontend/`. It's multi-brand: one codebase serves `sponge-kids`, `mnsa-safe`, and `mnsa-straight`, switched by hostname. (It used to be an Expo/React-Native app; that's gone.)

## Quick Commands
- Start everything (all 4 servers): `imbuo` from the workspace root
- Frontend only: `npm run dev --prefix frontend` — Vite dev server on port 5173, proxies `/api` → `http://127.0.0.1:8001`
- Build: `npm run build --prefix frontend` — runs `tsc -b && vite build`, outputs to `public/build/` with `base: '/build/'` so Laravel serves the SPA
- (No test runner in this package.)

## Key Files
- `frontend/src/main.tsx` — entry; mounts `<App/>`, imports `global.css`
- `frontend/src/App.tsx` — detects the brand (`detectBrand()`), sets `<html data-brand="…">` + `document.title`, imports each brand's `theme.css` (plus mnsa's `app.css`), and wraps the tree in `BrandContext` → `AuthProvider` → `AuthSync` → `BrowserRouter` → `AppRouter`. `AuthSync` re-verifies subscription on load via `fetchMe()`. Also bootstraps `_token` in `api.ts` synchronously from `localStorage.auth_token` at module load — without this, child useEffects (e.g. `Account`/`SkHome` calling APIs on mount) fire before `AuthSync`'s effect can sync the token, causing a 401 → auto-logout → redirect on hard refresh of any authed page.
- `frontend/src/router.tsx` — `AppRouter`: routes + guards. `RequireAuth` → `/login`; `RequireSubscription` → `/subscribe`; `RedirectIfAuthenticated` → `brand.postAuthRoute`. `/` → `SkSplash` if `brand.hasSearch` else `MnsaSplash`; `/login`, `/register`, `/subscribe` (shared); sponge-kids-only `/home`, `/reports/:id`; mnsa-only `/extension`; `*` → `NotFound`.
- `frontend/src/shared/brand-context.ts` — `BrandKey` (`'sponge-kids' | 'mnsa-safe' | 'mnsa-straight'`), `BrandConfig` (feature flags `hasSearch`/`hasReports`/`hasExtensionPage`/`postAuthRoute`), `detectBrand()`, `useBrand()`, `isMnsa()`
- `frontend/src/shared/auth-context.tsx` — `AuthProvider` / `useAuth()`; token + user + subscription flag persisted in `localStorage`; `login`/`logout`/`setSubscribed`
- `frontend/src/shared/api.ts` — fetch wrapper; `setApiToken`, `setApiBrand` (adds `X-Brand` header), `setOnUnauthorized` (401 auto-logout); `apiLogin`/`apiRegister`/`apiLogout`, `fetchMe`, `fetchReports`, `fetchBillingStatus`, `fetchCheckoutUrl`
- `frontend/src/shared/useExtensionState.ts` — `useExtensionState()` hook returning `'unknown' | 'not-installed' | 'installed' | 'open'` for the MNSA Chrome extension. Reads `document.documentElement.dataset.mnsaExtension*` stamps synchronously on mount, then listens for `window.postMessage({ source: 'imbuo-mnsa-ext', type: 'state', ... })` updates pushed by the extension's content script. Times out unknown→not-installed at 1.5s.
- `frontend/src/shared/pages/` — `Login.tsx`, `Register.tsx`, `Subscribe.tsx` (RevenueCat: opens `checkout_url` in a tab, polls billing status), `NotFound.tsx`
- `frontend/src/global.css` — brand-neutral utility classes (`page-center`, `auth-card`, `auth-title`, `auth-subtitle`, `form-input`, `form-label`, `btn-primary`, `btn-secondary`, `btn-link`) — all consume `var(--color-*)`
- `frontend/src/brands/sponge-kids/theme.css`, `frontend/src/brands/mnsa/theme.css` — per-brand CSS-var tokens under a `data-brand` selector. sponge-kids: `:root[data-brand="sponge-kids"]`; mnsa: `:root[data-brand^="mnsa"]` (a prefix match, so `mnsa-safe` and `mnsa-straight` share identical tokens)
- `frontend/src/brands/sponge-kids/pages/` — `Splash.tsx`, `Home.tsx` (search-first report list, debounced), `Report.tsx`
- `frontend/src/brands/mnsa/` — `pages/Splash.tsx` (a redirect — the `/` landing is now server-rendered Blade, see `resources/views/landing/`), `pages/Extension.tsx` (post-subscribe screen, switches between three states from `useExtensionState`: install CTA / "open it from your toolbar" hint / "Continue in the sidebar →" with an animated arrow), `theme.css`, `app.css` (overrides of the shared button/card/form classes, scoped `:root[data-brand^="mnsa"]`; also hosts the `.ext-*` arrow-animation classes for the Extension page and the `.mnsa-auth-page` / `.auth-shell` / `.auth-hero` / `.auth-eyebrow` / `.auth-display-title` classes for the MNSA login/register hero treatment)

## Non-Obvious Patterns
**Multi-brand by hostname.** `detectBrand()` maps `sponge.kids` → `sponge-kids`, `makenetflixsafeagain.com` → `mnsa-safe`, `makenetflixstraightagain.com` → `mnsa-straight`. In dev, `?brand=<key>` overrides and is remembered in `sessionStorage`. `App.tsx` puts `data-brand` on `<html>`; theme tokens key off it; `BrandConfig` feature flags drive routing. (Mirrors the backend's `config/brands.php` + `IdentifyBrand` middleware.)

**`mnsa-safe` and `mnsa-straight` share everything but copy.** Same theme tokens (the `theme.css` selector is `[data-brand^="mnsa"]`), same components under `brands/mnsa/`. Per-brand copy for the landing lives in `config/mnsa_landing.php` — not in the SPA.

**Route gating, not screen gating.** Auth/subscription gates live in `router.tsx` (`RequireAuth`, `RequireSubscription`), not inside screens. Logged-out → `/login`; logged-in-but-unsubscribed → `/subscribe`; the landing page and auth pages are public.

**Auth state in localStorage, re-verified on load.** Token, user object, and subscription flag persist in `localStorage`. On app load `AuthSync` calls `fetchMe()` to re-confirm the subscription every time (deliberate — avoids a cross-brand `localStorage` leak in dev).

**API client sends `X-Brand` and auto-logs-out on 401.** `setApiBrand` adds an `X-Brand` header so the backend resolves the right brand in dev (where `Host` doesn't match a prod domain). `setOnUnauthorized` clears auth state on any 401 — no token-refresh logic.

**Styling is plain CSS files + inline styles — no CSS-in-JS, no CSS modules.** SPA layers: `global.css` (shared utility classes) → per-brand `theme.css` (CSS-var tokens) → per-brand stylesheets (`brands/mnsa/app.css`). Components also use inline `style={{…}}` liberally. MNSA overrides of the shared classes are scoped under `:root[data-brand^="mnsa"]` so sponge-kids is unaffected. The MNSA landing has its own separate stylesheet at `public/css/mnsa-landing.css` (static; served directly by Laravel, not bundled by Vite).

**Chrome-extension detection on `/extension` is a cross-repo handshake.** The sibling repo `imbuo-netflix-chrome` ships a content script (`content/mnsa-detect.js`) that the extension injects on `localhost:5173/*` and the two MNSA prod domains at `document_start`. That script (a) stamps `data-mnsa-extension-installed="1"` / `data-mnsa-extension-panel-open="0|1"` on `<html>` for synchronous detection, and (b) posts `{ source: 'imbuo-mnsa-ext', type: 'state', installed, panelOpen }` via `window.postMessage` whenever state changes. The hook (`useExtensionState`) reads the stamps on mount, listens for postMessage updates, and times out to `'not-installed'` if no message arrives within 1.5s. Panel open/closed is tracked extension-side via a long-lived `chrome.runtime.connect({ name: 'mnsa-panel' })` port from `panel/panel.js`; its `onConnect`/`onDisconnect` is the open/close signal, broadcast back to all matching tabs. If you add new MNSA domains, the extension's `manifest.json` (`host_permissions` + the three `content_scripts` blocks for `auth-bridge.js` / `auth-relay.js` / `mnsa-detect.js`) and `background.js`'s `MNSA_PARENT_TAB_MATCHES` constant must be updated in lockstep.

**The MNSA `/` landing is server-rendered Laravel Blade** (`resources/views/landing/mnsa.blade.php` + partials, `LandingController`, copy lives in `config/mnsa_landing.php`) — not the React SPA. The SPA handles `/login`, `/register`, `/subscribe`, `/extension`, and the whole sponge-kids site. The landing's app-route links (`/register`, `/login`) are built from `config('app.frontend_url')` (= `FRONTEND_URL`): empty in prod (relative — Laravel serves the SPA same-origin), set to the Vite URL in local dev, where they're absolute and carry `?brand=` (the SPA on a different origin can't read the host for brand detection).
